## Notes

- Dashboard surfaces should feel operational, dense enough for repeat use, and calm.
- Prioritize scanability over marketing drama.

## Implementation Ideas

- Standardize panels, data cards, filters, and empty states through shared UI primitives.
- Use selected states and score surfaces consistently.

## Performance Considerations

- Avoid animated charts unless they clarify state.
- Keep table and card lists server-rendered where practical.

## Adaptation Ideas For MatchIQ

- Position scores as intelligence signals, with clear explanations and next actions.
